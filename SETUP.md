## Environment Setup

### 1. Clone the repository

```bash
git clone <repository-url>
cd spring-lessons
```

### 2. Install pre-commit hooks

```bash
pre-commit install
```

### 3. Start the infrastructure

Start all external services (database, Kafka, Keycloak, observability stack):

```bash
podman compose \
  --project spring-lessons \
  --file ./collections/compose-env.yaml \
  up --force-recreate --remove-orphans --detach
```

This starts: PostgreSQL, Kafka, Keycloak, Mailpit, Wiremock, OTel Collector, Jaeger, Prometheus, Grafana, Loki.

### 4. Verify services are up

| Service | URL |
| --- | --- |
| Keycloak | <http://localhost:8080> |
| Jaeger UI | <http://localhost:16686/search> |
| Prometheus | <http://localhost:9090> |
| Grafana | <http://localhost:3000> |
| Mailpit UI | <http://localhost:8025> |
| Wiremock admin | <http://localhost:9998/__admin/> |

## Running the Application

### Option A — VS Code (recommended)

Use the launch configuration **"Run Spring Lessons"** (`.vscode/launch.json`).  
It automatically:

1. Runs the `start-podman-env` pre-launch task (cleans up, starts containers, compiles)
2. Starts the application with the correct Spring profile
3. On stop, runs `stop-podman-env` (stops containers, exports Keycloak realm)

**Spring profiles applied:**

- Linux / macOS: `default,linux`
- Windows: `default,windows`

### Option B — Maven CLI

```bash
# Compile
mvn compile --file pom.xml --settings settings.xml

# Run (Linux / macOS)
mvn spring-boot:run --file pom.xml --settings settings.xml \
  -Dspring-boot.run.profiles=default,linux

# Run (Windows)
mvn spring-boot:run --file pom.xml --settings settings.xml \
  -Dspring-boot.run.profiles=default,windows
```

### Application endpoints

Once started:

| Resource | URL |
| --- | --- |
| Application base | <http://localhost:8888/spring-app> |
| Swagger UI | <http://localhost:8888/spring-app/swagger-ui/index.html> |
| Scalar UI | <http://localhost:8888/spring-app/scalar> |
| Actuator | <http://localhost:8889/actuator> |
| Database | `jdbc:postgresql://localhost:5432/spring` |
| Kafka broker | `localhost:29092` |

### Active Spring profiles

| Profile | File | Purpose |
| --- | --- | --- |
| `default` | `application.properties` | Main configuration |
| `linux` / `windows` | `application-runtime.properties` | OS-specific overrides (paths, etc.) |

## Running Tests

Tests require the infrastructure to be running first (see [Environment Setup](#environment-setup)).

```bash
# Run all tests
mvn test --file pom.xml --settings settings.xml
```

Coverage reports are generated by JaCoCo after the test phase.  
Excluded from coverage: mapper, config, model, and main application class.

### Full test cycle (via VS Code task)

The `test-application` task performs the full cycle:

1. Remove existing containers and volumes
2. Clean up `bin/` and `logs/` folders
3. Start fresh environment containers
4. Run `mvn clean package`

## Build & Packaging

### Build the JAR

```bash
# With tests (Linux / macOS)
mvn package --file pom.xml --settings settings.xml \
  -Dspring.profiles.active=default,linux

# Skip tests
mvn package --file pom.xml --settings settings.xml \
  -Dspring.profiles.active=default,linux -DskipTests=true
```

Output JAR is placed in `./bin/`.

### Build the container image

```bash
podman build --rm=true --file Containerfile --tag spring-lessons:latest .
```

The VS Code task `podman-build-image` handles this automatically (runs `mvn clean package -DskipTests` first).

### Run as container (full stack)

```bash
podman compose \
  --project spring-lessons \
  --file ./collections/compose-env.yaml \
  --file ./collections/compose-app.yaml \
  up --force-recreate --remove-orphans --detach
```

## VS Code Tasks Reference

Tasks are defined in `.vscode/tasks.json`.

### Lifecycle tasks

| Task | Description |
| --- | --- |
| `start-podman-env` | Clean up, start containers, compile sources |
| `stop-podman-env` | Stop containers, remove volumes, export Keycloak realm |
| `test-application` | Full test cycle (clean env → start → package + test) |
| `cleanup-env` | Full Podman cleanup (containers, images, volumes, system prune) |

### Build tasks

| Task | Description/Command |
| --- | --- |
| `compile-source-code` | `mvn clean compile` |
| `mvn-clean` | `mvn clean` |
| `mvn-compile` | `mvn compile` |
| `mvn-package` | `mvn package` (with tests) |
| `mvn-package-skip-tests` | `mvn package -DskipTests=true` |
| `mvn-test` | `mvn test` |
| `mvn-spotless-apply` | Apply Google Java code formatting |
| `podman-build-image` | Build the container image |

### Container tasks

| Task | Description |
| --- | --- |
| `podman-compose-env-up` | Start environment services |
| `podman-compose-env-stop` | Stop environment services |
| `podman-compose-app-up` | Start environment + app container |
| `podman-compose-app-stop` | Stop environment + app container |

### Utility tasks

| Task | Description |
| --- | --- |
| `mvn-dependencies-tree` | Output dependency tree to `logs/maven-dependency-tree.log` |
| `mvn-display-dependency-updates` | Output available dependency updates to `logs/` |
| `mvn-display-plugin-updates` | Output available plugin updates to `logs/` |
| `pre-commit-autoupdate` | Update pre-commit hook versions |
| `pre-commit-run-all-files` | Run all pre-commit checks on every file |
| `podman-compose-keycloak-export` | Export Keycloak realm configuration |

## Useful Commands

### Podman — stop and clean up everything

```bash
podman container rm --force --all
podman volume rm --force --all
podman system prune --force --all --volumes
```

### Maven — code formatting

```bash
mvn spotless:apply --file pom.xml --settings settings.xml
```

### Maven — check dependency updates

```bash
mvn versions:display-dependency-updates --file pom.xml --settings settings.xml
mvn versions:display-plugin-updates --file pom.xml --settings settings.xml
```

### Export Keycloak realm

```bash
podman compose \
  --project spring-lessons \
  --file ./collections/compose-keycloak-export.yaml \
  up --abort-on-container-exit --remove-orphans
```

### Postman collection

Import `./collections/postman.json` into Postman for a ready-made request collection covering all API endpoints.
